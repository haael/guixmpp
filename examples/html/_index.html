HTML Tag Examples

  • a: The <a> tag defines a hyperlink.
  • abbr: The <abbr> tag defines an abbreviated form of a longer word or phrase.
  • address: The <address> tag specifies the author's contact information.
  • area: The <area> tag defines a specific area within an image map.
  • article: The <article> tag defines an article.
  • aside: The <aside> tag defines some content loosely related to the page content.
  • audio: The <audio> tag embeds a sound or an audio stream in an HTML document.
  • b: The <b> tag displays text in a bold style.
  • base: The <base> tag defines the base URL for all relative URLs in a document.
  • bdi: The <bdi> tag represents text that is isolated from its surrounding for the purposes of bidirectional text formatting.
  • bdo: The <bdo> tag overrides the current text direction.
  • blockquote: The <blockquote> tag represents a section that is quoted from another source.
  • br: The <br> tag produces a single line break.
  • button: The <button> tag creates a clickable button.
  • canvas: The <canvas> tag defines a region in the document for drawing graphics on the fly via scripting.
  • caption: The <caption> tag defines the caption or title of the table.
  • cite: The <cite> tag indicates a citation or reference to another source.
  • code: The <code> tag specifies text as computer code.
  • col: The <col> tag defines attribute values for one or more columns in a table.
  • colgroup: The <colgroup> tag specifies attributes for multiple columns in a table.
  • data: The <data> tag links a piece of content with a machine-readable translation.
  • datalist: The <datalist> tag represents a set of pre-defined options for an <input> element.
  • dd: The <dd> tag specifies a description or value for the term (<dt>) in a description list (<dl>).
  • del: The <del> tag represents text that has been deleted from the document.
  • details: The <details> tag represents a widget from which the user can obtain additional information or controls on-demand.
  • dfn: The <dfn> tag specifies a definition.
  • dialog: The <dialog> tag defines a dialog box or subwindow.
  • div: The <div> tag specifies a division or a section in a document.
  • dl: The <dl> tag defines a description list.
  • dt: The <dt> tag defines a term (an item) in a description list.
  • em: The <em> tag defines emphasized text.
  • embed: The <embed> tag embeds external applications, typically multimedia content like audio or video, into an HTML document.
  • fieldset: The <fieldset> tag specifies a set of related form fields.
  • figcaption: The <figcaption> tag defines a caption or legend for a figure.
  • figure: The <figure> tag represents a figure illustrated as part of the document.
  • footer: The <footer> tag represents the footer of a document or a section.
  • form: The <form> tag defines an HTML form for user input.
  • h1: The <h1> tag defines a top-level heading.
  • h2: The <h2> tag defines a second-level heading.
  • h3: The <h3> tag defines a third-level heading.
  • h4: The <h4> tag defines a fourth-level heading.
  • h5: The <h5> tag defines a fifth-level heading.
  • h6: The <h6> tag defines a sixth-level heading.
  • header: The <header> tag represents the header of a document or a section.
  • hr: The <hr> tag produces a horizontal line.
  • i: The <i> tag displays text in an italic style.
  • iframe: The <iframe> tag displays a URL in an inline frame.
  • img: The <img> tag represents an image.
  • input: The <input> tag defines an input control.
  • ins: The <ins> tag defines a block of text that has been inserted into a document.
  • kbd: The <kbd> tag specifies text as keyboard input.
  • label: The <label> tag defines a label for an <input> control.
  • legend: The <legend> tag defines a caption for a <fieldset> element.
  • li: The <li> tag defines a list item.
  • main: The <main> tag represents the main or dominant content of the document.
  • map: The <map> tag defines a client-side image-map.
  • mark: The <mark> tag represents text highlighted for reference purposes.
  • menu: The <menu> tag represents a list of commands.
  • meter: The <meter> tag represents a scalar measurement within a known range.
  • nav: The <nav> tag defines a section of navigation links.
  • noscript: The <noscript> tag defines alternative content to display when the browser doesn't support scripting.
  • object: The <object> tag defines an embedded object.
  • ol: The <ol> tag defines an ordered list.
  • optgroup: The <optgroup> tag defines a group of related options in a selection list.
  • option: The <option> tag defines an option in a selection list.
  • output: The <output> tag represents the result of a calculation.
  • p: The <p> tag defines a paragraph.
  • picture: The <picture> tag defines a container for multiple image sources.
  • pre: The <pre> tag defines a block of preformatted text.
  • progress: The <progress> tag represents the completion progress of a task.
  • q: The <q> tag defines a short inline quotation.
  • rp: The <rp> tag provides fall-back parentheses for browsers that don't support ruby annotations.
  • rt: The <rt> tag defines the pronunciation of characters presented in a ruby annotation.
  • ruby: The <ruby> tag represents a ruby annotation.
  • s: The <s> tag represents contents that are no longer accurate or no longer relevant.
  • samp: The <samp> tag specifies text as sample output from a computer program.
  • section: The <section> tag defines a section of a document.
  • select: The <select> tag defines a selection list within a form.
  • small: The <small> tag displays text in a smaller size.
  • source: The <source> tag defines alternative media resources for media elements like <audio> or <video>.
  • span: The <span> tag defines an inline styleless section in a document.
  • strong: The <strong> tag indicates strongly emphasized text.
  • sub: The <sub> tag defines subscripted text.
  • summary: The <summary> tag defines a summary for the <details> element.
  • sup: The <sup> tag defines superscripted text.
  • table: The <table> tag defines a data table.
  • tbody: The <tbody> tag groups a set of rows defining the main body of the table data.
  • td: The <td> tag defines a cell in a table.
  • textarea: The <textarea> tag defines a multi-line text input control.
  • tfoot: The <tfoot> tag groups a set of rows summarizing the columns of the table.
  • th: The <th> tag defines a header cell in a table.
  • thead: The <thead> tag groups a set of rows that describes the column labels of a table.
  • time: The <time> tag represents a time and/or date.
  • tr: The <tr> tag defines a row of cells in a table.
  • track: The <track> tag defines text tracks for media elements like <audio> or <video>.
  • u: The <u> tag displays text with an underline.
  • ul: The <ul> tag defines an unordered list.
  • var: The <var> tag defines a variable.
  • video: The <video> tag embeds video content in an HTML document.
  • wbr: The <wbr> tag represents a line break opportunity.

CSS Display Attribute Examples

  • inline: Displays an element as an inline element (like <span>).
  • block: Displays an element as a block element (like <p>).
  • contents: Makes the container disappear, making the child elements children of the element the next level up in the DOM.
  • flex: Displays an element as a block-level flex container.
  • grid: Displays an element as a block-level grid container.
  • inline-block: Displays an element as an inline-level block container.
  • inline-flex: Displays an element as an inline-level flex container.
  • inline-grid: Displays an element as an inline-level grid container.
  • inline-table: The element is displayed as an inline-level table.
  • list-item: Let the element behave like a <li> element.
  • run-in: Displays an element as either block or inline, depending on context.
  • table: Let the element behave like a <table> element.
  • table-caption: Let the element behave like a <caption> element.
  • table-column-group: Let the element behave like a <colgroup> element.
  • table-header-group: Let the element behave like a <thead> element.
  • table-footer-group: Let the element behave like a <tfoot> element.
  • table-row-group: Let the element behave like a <tbody> element.
  • table-cell: Let the element behave like a <td> element.
  • table-column: Let the element behave like a <col> element.
  • table-row: Let the element behave like a <tr> element.
  • none: The element is completely removed.
